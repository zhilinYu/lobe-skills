# 贡献指南

感谢你对 bookmark-manager-skill 的关注！

## 贡献方式

### 报告 Bug
1. 检查是否已有相关 Issue
2. 创建新 Issue 时请包含：
   - 问题描述
   - 复现步骤
   - 期望行为 vs 实际行为
   - 环境信息（macOS版本、LobeHub版本等）

### 提交功能请求
1. 创建 Feature Request Issue
2. 描述你想要的功能和使用场景
3. 如果可能，附上示例或参考

### 提交代码
1. Fork 本仓库
2. 创建特性分支：`git checkout -b feature/amazing-feature`
3. 提交更改：`git commit -m 'feat: add amazing feature'`
4. 推送分支：`git push origin feature/amazing-feature`
5. 创建 Pull Request

### 代码规范

#### 提交信息格式
使用 Conventional Commits：
- `feat:` 新功能
- `fix:` 修复
- `docs:` 文档更新
- `refactor:` 重构
- `style:` 格式调整
- `chore:` 杂项

#### Skill 编写规范
- 使用中文编写核心工作流说明
- Markdown 格式规范，保持结构清晰
- 每个文件不超过 500 行

## 开发流程
1. 阅读 README.md 和 SKILL.md 了解整体架构
2. 查看 CHANGELOG.md 了解版本变化
3. 在 examples/ 目录添加使用示例
4. 更新 SKILL.md 中的工作流说明
5. 提交 PR 等待 review

## 行为准则
- 尊重所有贡献者
- 建设性反馈
- 关注技术本身
